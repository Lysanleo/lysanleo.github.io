11. Chp8 输入/输出系统 [重点]

核心关联芝士论述: 为了保证高速的主机(主要是 CPU)和不同速度的外设之间的高效和可靠的交互, CPU 必需通过 I/O 接口与外设连接, 并且为了与 CPU 交互信息方便, 接口内部要放置一些 CPU 能直接访问的寄存器, 这些寄存器称为 端口 PORT

  • P241-242
    • 接口的作用
      命令口
      接口内用于接收来自 CPU 等主控设备的控制命令的寄存器称为命令端口
      状态口
      接口内用于向 CPU 报告 I/O 设备工作状态的寄存器称为状态端口或状态口
      数据口
      接口内在外设和总线间交换数据的缓冲寄存器称为数据端口或数据口
    • IO 接口与外设间的数据传送方式, 并注意什么时候使用什么方法:
      • 无条件传送方式: 速度极慢/简单的外围设备因其不能跟上时钟的节奏
      • 异步传送方式: 慢速/中速的外围设备
      • 同步传送方式: 高速的外围设备
  • P247 主要在讲接口与 CPU(主机)间的数据传送方式
    程序查询方式(又叫做程序控制 IO 方式, 轮查)
    是利用程序控制实现 CPU 和外部设备之间的数据传送
    中断
    是一种程序随机切换的方式, 有时也统称为异常
    中断应用
    中断方式的典型应用(教材有一张形象的图):
    1. 实现 CPU 与外界(外部设备)进行信息交换的握手联络
    2. 故障处理(针对故障事件)
    3. 实时处理(针对事件, 所以实时)
    4. 程序调度
    5. 软中断
  • P260 DMA DMA 控制器从 CPU 完全接管对总线的控制, 数据交换不经过 CPU, 而直接在内存和 I/O 之间进行.
    DMA 的基本概念
    DMA(直接内存访问), 是一种完全由硬件执行的 I/O 交换的工作方式(与程序查询形成对应).
    DMA 可以完成的四项基本操作
    由于 DMA 在讲一间内存和 外围设备 间的数据传送故事, DMA 过程要告诉 CPU 自己将接管总线, 然后, 外设可以发送 DMA 请求, DMA 要处理内存寻址的问题, 然后结束过程需要向 CPU 报告
    1. 从外围设备发出 DMA 请求
    2. CPU 响应请求
    3. 由 DMA 控制器对内存寻址
    4. 向 CPU 报告 DMA 操作的结束
    DMA 数据传送过程的 三个 阶段
    1. 传送前预处理
    2. 正式传送
    3. 传送后处理
  • P268 通道与其他芝士的粘连: DMA 控制器的出现已经减轻了 CPU 对数据输入输出的控制, 使得更多系统资源能够被其他任务使用 通道 则做了进一步的提升, 通道是一个 有特殊功能的处理器 它有自己的指令来专门负责数据输入输出的 传输控制, 因此 CPU 只需把传输下放通道, 然后负责 数据处理 就行了.
    • 通道的 五项任务: 通道已经确认为是对数据传输进行控制, 所以它与外围 I/O 设备间具有连接, 它也直接受主机(CPU)控制, 所以它能接收 CPU 的 I/O 指令, 从存储器选择属于某通道程序的通道指令, 进行译码并发送给各种 IO 控制模块. 组织外设和储存器进行数据传送, 并提供缓冲. 及时向 CPU 报告通道/外设的中断请求.
      1. 接收 CPU 的 I/O 指令…
      2. 从储存器选取属于该通道程序的的通道指令…
      3. 组织外设和存储器之间进行数据传送…
      4. 从外围设备得到设备的状态信息…
      5. 将外设的中断请求和通道本身的中断请求, 按次序及时报告给 CPU…
    • 通道的分类
      1. 选择通道(高速通道)
      2. 多路通道
        • 数组多路通道
        • 字节多路通道
